Running the model

Finally, when all prequisites are installed, the data is preprocessed, and the model is configured, we can run the model!

run.py

GEB aims to simulate both environment, for now the hydrological system, the behaviour of people and their interactions at large scale without sacrificing too much detail.

usage: run.py [-h] [--headless] [--no-browser] [--port PORT] --scenario
              SCENARIO [--switch_crops] [--gpu_device GPU_DEVICE]
              [--profiling] [--GPU] [--config CONFIG]
-h, --help

show this help message and exit

--headless

The model can be run with or without a visual interface. The visual interface is useful to display the results in real-time while the model is running and to better understand what is going on. You can simply start or stop the model with the click of a buttion, or advance the model by an x number of timesteps. However, the visual interface is much slower than running the model without it.

--no-browser

Do not open browser when running the model. This option is, for example, useful when running the model on a server, and you would like to remotely access the model.

--port <port>

Port used for display environment (default: 8521)

--scenario <scenario>

Here you can specify which scenario you would like to run. Currently 4 scenarios (base, self_investement, ngo_training, government_subsidies) are implemented, and model spinup are implemented.

--switch_crops

Whether agents should switch crops or not.

--gpu_device <gpu_device>

Specify the GPU to use (zero-indexed).

--profiling

Run GEB with with profiling. If this option is used a file profiling_stats.cprof is saved in the working directory.

--GPU

Whether a GPU can be used to run the model. This requires CuPy to be installed.

--config <config>

Path of the model configuration file.
